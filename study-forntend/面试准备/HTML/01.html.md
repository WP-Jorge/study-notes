- 渐进增强和优雅降级之间的区别

```text
1、渐进增强：针对低版本浏览器进行构建页面。保证最基本的功能，然后再针对高级浏览器进行效果、交互等改进和追加功能达到更好的用户体验
2、优雅降级：一开始就构建完整的功能，然后再针对低版本浏览器进行兼容
3、区别：
    (1) 优雅降级是从最复杂的现状开始，并试图减少用户体验的供给，而渐进增强则是从一个非常基础的、能够起作用的版本，并不断扩充，以适应未来环境的需求，
    (2) 优雅降级认为应该针对那些高级、完善的浏览器来设计网站，将过时、功能缺失的浏览器的测试工作安排到测试的最后阶段，并把测试对象限定为主流浏览器的前一个版本
```

- DOCTYPE(文档类型) 的作用

```text
1、原理：
    (1) DOCTYPE 是 HTML5 中一种标准通用标记语言的文档类型声明，目的在于告诉浏览器应该以什么样（html、xml）的文档类型定义来解析文档，不容的渲染模式会影响 CSS 甚至 JS 脚本的解析，必须声明再第一行
2、模式：
    (1) 默认模式，浏览器使用 W3C 的标准解析渲染页面。在标准模式中，浏览器以其支持的最高标准呈现页面
    (2) 怪异模式：浏览器使用自己的怪异模式解析渲染页面。在页面中。页面以一种比较宽松的向后兼容的形式显示
    (3) 区别：
        ① 盒模型：在 W3C 标准中，如果设置一个元素的宽高，指的是元素内容的宽高，在怪异模式下，宽高包含了 padding、border
        ② 设置行内宽高：标准模式下，给行内元素设置宽高无效，在怪异模式下生效
        ③ 设置百分比高度：标准模式下，一个元素的高度是由其包含的内容决定的，如果父元素没有设置高度，子元素设置百分比高度无效；使用 margin: 0 auto; 设置水平居中，而 怪异模式下失效
```

- Canvas和SVG的区别

```text
1、是什么：
    (1) SVG：
        ① 可伸缩矢量图形（Scalable Vector Graphics），是一种使用 XML 描述 2D 图形额语言
        ② 基于 XML，这意味着 SVG DOM 中的每个元素都是可用的，可以为每个元素添加 JS 事件
        ③ 在 SVG 中每个绘制的图形均视为对象，如果 SVG 对象属性发生变化，那么浏览器能够自动重现图形
    (2) Canvas：
        ① 通过 JS 来绘制 2D 图形
        ② 基于像素进行渲染
        ③ 一旦图形被绘制完成，他就不会继续得到浏览器的关注，如果其位置发生变化，那么整个场景也需要重新绘制，包括任何或许已被图形覆盖的对象
2、不同点：
    (1) SVG：
        ① 不依赖分辨率
        ② 支持事件处理器
        ③ 最适合带有大型渲染区域的应用程序
        ④ 复杂度高会减慢渲染速度
        ⑤ 不适合游戏应用
    (2) Canvas：
        ① 依赖分辨率
        ② 不支持事件处理
        ③ 弱的文本渲染能力
        ④ 能以 .png、.jpg 格式保存结果图像
        ⑤ 最适合图像密集型的游戏，其中许多对象会被频繁重绘
```

- 常见的meta标签有哪些

```text
1、是什么：
    (1) meta 标签是由 name 和 content 属性定义，用来描述网页文档属性，比如网页作者、网页描述、关键词等，除了 HTTP 标准规定的一些 name 可以使用，开发者还可自定义 name
2、常见的 meta 标签：
    (1) charset：用来描述 HTML 文档的编码类型：
        ① <meta charset="UTF-8" >
    (2) keywords：页面关键字：
        ① <meta name="keywords" content="关键字"/>
    (3) description：页面描述：
        ① <meta name="description" content="页面描述内容"/>
    (4) refresh：页面重定向和刷新：
        ① <meta http-equiv="refresh" content="0;url"/>
    (5) viewport：适配移动端，可以控制视口的大小和比例：content 参数：
        ① <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1"/>
        ② width：宽度(数值/device-width)
        ③ height：高度(数值/device-height)
        ④ initial-scale：初始缩放比例
        ⑤ maximum-scale：最大缩放比例
        ⑥ minimum-scale：最小缩放比例
        ⑦ user-scalable：是否允许用户缩放(yes/no)
    (6) robots：搜索引擎索引方式，content 参数：
        ① <meta name="robots" content="index,follow"/>
        ② all：文件将被索引，且页面上的链接可以被查询
        ③ none：文件不被索引，且页面上的链接不可以被查询
        ④ index：文件将被索引
        ⑤ follow：页面上的链接可以查询
        ⑥ noindex：文件不被索引
        ⑦ nofollow：页面上的链接不被查询
```

- 单页应用和多页应用

```text
1、单页面应用：
    (1) 每次页面跳转不会出现新的 HTML 文档，但是页面会跟着变化。原因是 JS 会感知 URL 的变化，感知变化后将原来的页面内容清除，然后挂载上下一个页面的内容。单页面应用是通过 JS 渲染来跳转页面的
    (2) 优点：页面切换快，不需要加载 HTML 文件，减少 HTTP 请求，提高性能
    (3) 缺点：首屏空白时间长，因为需要发送一个 HTTP 请求 和 JS 请求，当两个两个请求同时返回，首屏才会显示，SEO 效果差
2、多页面应用：
    (1) 每一次页面跳转，后台就会返回一个新的 HTML 文档。多页面应用是通过 HTTP 请求来实现页面跳转的
    (2) 优点：首屏加载时间快，SEO 效果好
    (3) 缺点：页面切换慢，因为每次切换页面都要发送 HTTP 请求，如果网速很慢，则页面跳转会出现卡顿的现象
```

- 超链接target属性的取值和作用？

```text
1、指定所链接页面在浏览器窗口中的打开方式
    (1) _self：a 标签的默认值，不保留原有页面，在这个窗口重新打开新的链接页面
    (2) _blank：保留原有的页面，开启一个新的窗口展示新页面
    (3) _parent：在父级窗口打开链接，如果含有该链接的框架不是嵌套的，则在浏览器全屏窗口中载入链接的文件，如 _self
    (4) _top：在当前的整个浏览器窗口中打开所链接的文档，会删除所有框架
    (5) _frameName：在指定窗口打开页面
```

- iframe优缺点
```text
1、优点：
    (1) iframe 能够原封不动的把嵌入的网页展示出来
    (2) 如果有多个网页引用 iframe，那么只需要修改 iframe 的内容，就可以实现每一个页面的修改，方便快捷
    (3) 网页如果为了统一风格，头部和尾部都是一样的，就可以写成一个页面，用 iframe 嵌套，可以增加代码的复用性
    (4) 遇到加载缓慢的第三方内容如图标或广告，可使用 iframe 解决
    (5) 重载页面时无需重载整个页面，只需要重载页面中的一个框架页
2、缺点：
    (1) iframe 会阻塞主页面的 onload 事件
    (2) 搜索引擎无法检索这种页面，不利于 SEO
    (3) iframe 和主页面共享连接池，而浏览器相对于同域的链接没有限制，所以会影响页面的并行加载
```
- src与href的区别
```text
1、src 与 href 都是用来引用外部资源
2、区别：
    (1) src：表示对资源的引用，它指向的内容会嵌入到当前标签所在的位置，src 会将其指向的资源下载并应用到文档中，如请求 JS 脚本。当浏览器解析到该元素的时候，会暂定其他资源的下载 和处理，直到该资源加载、编译、执行完毕，所以一般 JS 脚本放在最后面
    (2) href：表示超文本引用，它指向一些网络资源，建立和当前元素或文档的链接关系，当浏览器识别到它指向的文件时，会并行下载资源，不会停止对当前文档的处理，常用在 link、a 上
```
- title与h1的区别、b与strong的区别、i与em的区别
```text
1、title 属性没有明确意义，只表示是个标题，h1 则表明层次明确的标题，对页面信息的抓取有很大的影响
2、strong 标签具有语义，起到加强语气的作用，而 b 标签没有，b 标签只是一个简单的加粗标签，搜索引擎更侧重于 strong
3、i 内容为斜体，em 表示强调的文本
```
